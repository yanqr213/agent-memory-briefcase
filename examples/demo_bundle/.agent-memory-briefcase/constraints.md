# Constraints

## Hard Constraints
- Keep the tool offline-first and avoid external service dependencies.
- Preserve mirrored JSON and Markdown records for durable project memory.

## Taboos
- Do not store secrets or private credentials in memory artifacts.
- Do not rely on generated briefs as the only source of project truth.

## Risk Hotspots
- Ownership and test evidence can become stale quickly in active repositories.
- Overly long briefs can crowd out recent decisions and delivery context.
